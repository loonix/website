## 🎮 Quick Start Guide

### Step 1: Get GitHub Token (1 min)
```
https://github.com/settings/tokens
→ Generate new token (classic)
→ Scopes: repo, user:email, read:org
→ Copy token (ghp_...)
```

### Step 2: Configure (30 seconds)
Edit `designer-mode.js` line 57:
```javascript
this.githubAuth = new GitHubAuth({
  owner: 'YOUR_USERNAME',
  repo: 'YOUR_REPO_NAME'
});
```

### Step 3: Push to GitHub (2 min)
```bash
git init
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git add .
git commit -m "Initial CMS setup"
git push -u origin main
```

### Step 4: Start Editing! (Right now!)
```bash
python3 -m http.server 8000
# Open http://localhost:8000
# Press Ctrl+Shift+E
# Click GitHub button
# Enter token
# Start creating! 🎨
```

## 🧠 How It Thinks (BDI + OODA)

### BDI Architecture
```
BELIEFS: System monitors current state
  ├─ GitHub: commits, SHAs, branches
  ├─ Local: localStorage, edits, changes
  ├─ Network: online, latency, API status
  └─ Activity: user actions, idle time

DESIRES: System goals
  ├─ Keep content synchronized
  ├─ Avoid data loss
  ├─ Prevent conflicts
  └─ Optimize performance

INTENTIONS: System actions
  ├─ Auto-sync when safe
  ├─ Prompt user on conflicts
  ├─ Queue operations when offline
  └─ Optimize images before upload
```

### OODA Loop (Runs Every 30s)
```
OBSERVE → Detect changes in environment
ORIENT  → Assess context and conflicts
DECIDE  → Choose optimal action
ACT     → Execute with error handling

→ Repeat continuously
```
